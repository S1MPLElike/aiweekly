AI 智能工作周报系统
项目需求说明书（完整版・Word 正式版）
适用开发：Vue3 + Go Gin + MySQL + LLM
开发模式：前后端分离、按模块逐步实现
核心定位：每日工作记录 → AI 生成日报 → 日历台账管理 → 按周生成周报 → AI 工作建议
周规则：系统固定以 周一 至 周日 为一个完整周
1 项目概述
本项目是面向职场人、实习生、基层员工的 PC 端网页工作记录与 AI 汇报工具。用户可每日记录工作内容，系统基于真实工作记录，通过 LLM 大模型自动生成规范、专业的日报与周报。系统提供日历式台账，支持查看、编辑任意历史日期的工作记录，并可根据用户历史数据提供 AI 工作建议。所有数据（工作记录、日报、周报）均持久化存储，保证安全、可追溯、可复盘。
本系统采用前后端分离架构，前端使用 Vue3 构建交互界面，后端使用 Go + Gin 提供稳定高效的接口服务，数据库使用 MySQL，登录鉴权使用 JWT。

AI 工作建议
3 页面结构
登录成功后进入主页面，采用 左侧固定菜单 + 右侧内容区域 布局。
左侧菜单（从上到下固定顺序）：
昨日工作
开启今日工作（核心）
月台账（日历）
设置
我（AI 工作建议）
右侧区域根据菜单切换展示对应模块。
4 功能模块详细说明
4.1 登录模块
用户名、密码登录
JWT 身份验证
登录成功进入主页
所有业务接口必须携带 JWT 访问
登录失败给出明确提示
4.2 昨日工作
进入主页默认展示
展示当前用户前一天的所有工作记录
只读展示，不可编辑、删除
列表形式逐条展示
4.3 开启今日工作（核心功能）
自动显示当前日期
支持添加多条今日工作事件
支持对已存在事件进行修改、删除
提供 生成今日工作报告 按钮
支持两种报告类型：极简版、默认版
点击后调用 LLM 生成日报
生成内容自动保存至数据库
再次进入页面自动回显当天已生成报告
4.4 月台账（日历视图）
展示整月日历，一行代表一周
日历排列固定：周一～周日
点击任意日期弹出弹窗，展示当天工作记录
弹窗支持：查看、新增、编辑、删除历史记录
日历每一行最右侧放置按钮：查看本周周报
点击自动获取该行对应 周一～周日 的全部记录
调用 LLM 生成周报并保存
周报记录：周一日期、周日日期、周报内容
4.5 设置
修改用户昵称
修改登录密码
退出登录（清除 JWT，返回登录页）
4.6 我（AI 工作建议）
展示当前用户基本信息
提供按钮：获取 AI 工作建议
调用 LLM，基于历史工作记录生成建议
页面直接展示建议内容
5 LLM 接口说明（必须实现 3 个）
所有接口由后端统一封装，前端不直接调用大模型。
5.1 生成今日日报
输入：当天所有工作记录、报告类型（极简 / 默认）、日期
输出：结构化、通顺、可直接提交的日报文本
存储：保存至 daily_reports 表
5.2 生成周报
输入：整周（周一～周日）所有工作记录、周开始、周结束
输出：完整、专业、适合提交领导的周报
存储：保存至 weekly_reports 表
5.3 AI 工作建议
输入：用户近期工作记录
输出：工作改进、效率提升、职业习惯建议
存储：无需存表，直接返回展示
6 数据库表结构（共 4 张）
6.1 users（用户表）
id BIGINT 主键
username VARCHAR (50) 用户名
password VARCHAR (255) 密码（密文）
name VARCHAR (50) 昵称
created_at DATETIME
updated_at DATETIME
6.2 work_records（每日工作记录表）
id BIGINT 主键
user_id BIGINT 用户 ID
content TEXT 工作内容
record_date VARCHAR (10) 日期（YYYY-MM-DD）
created_at DATETIME
updated_at DATETIME
6.3 daily_reports（日报表）
id BIGINT 主键
user_id BIGINT
content TEXT AI 生成日报
type VARCHAR(20) simple/default
report_date VARCHAR(10)
created_at DATETIME
6.4 weekly_reports（周报表）
id BIGINT 主键
user_id BIGINT
content TEXT AI 生成周报
week_start VARCHAR (10) 周一日期
week_end VARCHAR (10) 周日日期
created_at DATETIME
7 后端开发规则
一周固定为：周一 至 周日
按周查询必须按周一到周日获取数据
日报、周报生成后必须保存到数据库
所有接口遵循 RESTful 规范
统一返回格式：code、msg、data
JWT 全局鉴权
8 前端开发规则
左侧菜单切换使用路由，无刷新
日历以周一为起始
历史记录编辑使用弹窗（Dialog）
所有请求统一携带 JWT
界面风格简洁、清爽、职场工具风
9 推荐开发顺序
创建数据库与 4 张表
后端：登录接口 + JWT
前端：登录页 + 主页布局
工作记录增删改查接口
昨日工作、今日工作页面
LLM 生成日报接口
月台账日历页面
按周查询 + 周报生成接口
历史记录弹窗编辑功能
设置、AI 工作建议模块
整体联调、修复、完善
